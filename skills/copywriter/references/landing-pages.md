# Marketing Landing Pages

For product / app marketing pages: hero, headline + subhead, feature sections,
social proof, FAQ, CTAs. Read SKILL.md first — the core principles all apply.

A landing page has one job: get the visitor to take the next step (download,
sign up, start trial). Every section either earns the next scroll or wastes
the visitor's time.

## The Above-the-Fold Test

The visitor decides whether to keep scrolling within ~3 seconds. Above the
fold needs to answer:

1. **What is this?** (one line, plain language)
2. **Who is it for?** (implicit or explicit)
3. **Why should I care?** (the outcome — what changes for me)
4. **What do I do next?** (one clear CTA)

If any of those four are missing or unclear above the fold, the page is
broken.

## Hero Section

```
[ Headline — the outcome, in user language, ≤10 words ]
[ Subhead — one sentence, 12–20 words, makes the headline concrete ]
[ Primary CTA ]   [ Optional secondary action ]
[ Visual proof: product shot / video / screenshot ]
```

### Headline rules

- **Lead with the outcome, not the category.** "Never lose another workout"
  beats "The smart workout tracker".
- **≤10 words.** If it doesn't fit, it isn't sharp enough yet.
- **No "Welcome to..." / "Introducing..." / "Meet ___".**
- **Pass the hand test.** Cover the visual. Does the headline stand on its own?

### Subhead rules

- **Make the headline concrete.** If the headline is the promise, the subhead
  is the proof / mechanism / specificity.
- **One sentence.** Don't write a paragraph.
- **Names the user, the problem, or the mechanism** — at least one.

### Hero examples

**Generic / weak**
> The smart workout tracker.
> Track your workouts and reach your goals with our powerful app.

**Specific / strong**
> Never repeat last week's workout by accident.
> GainsLog logs every set in two taps and tells you exactly when to add weight.

## Social Proof (right after the hero, if you have it)

If it's real, put it high. Visitors trust other users more than they trust you.

- **Numbers**: "10,000 lifters log every workout in GainsLog."
- **Press logos**: only if they're recognizable to the audience.
- **Quotes**: short, specific, with name + context. Avoid generic praise.
- **Ratings**: only if 4.5+ and from a venue the visitor cares about.

Don't fake it. Don't include "as featured in" with logos that just mentioned
your tweet once.

## Feature Sections

The body of the page is usually 3–5 feature sections, each one alternating
left/right with an image.

Each section = **one outcome**, supported by **one feature**, **one image**.

Pattern per section:
```
[ Section headline — one outcome, ≤8 words ]
[ Body — 1–3 sentences, in user language, says how this changes their day ]
[ Optional micro-CTA / inline link ]
[ Visual: product screenshot showing the feature ]
```

### Section headline rules

- Same as App Store screenshots: outcome-first, ≤8 words, hand test passes.
- Don't repeat the hero promise. Each section should add a new angle.
- Don't title sections "Features", "Benefits", "How it works" — those are
  category labels, not headlines. Lead with the outcome.

### Bad → Good

| Bad                              | Good                                          |
|----------------------------------|-----------------------------------------------|
| Powerful workout tracking        | Log a set in two taps.                        |
| Smart progression algorithm      | Know exactly when to add weight.              |
| Cross-device sync                | Open it on your phone, finish it on your iPad.|
| Beautiful interface              | Built to use mid-set, with one hand.          |

## CTAs Throughout the Page

- **Primary CTA repeats every 1–2 sections.** Don't make the visitor scroll
  back up.
- **CTA labels are verbs + outcomes**, not categories.
  - Bad: "Get started" / "Sign up" / "Download"
  - Good: "Start tracking workouts" / "Try GainsLog free" / "Get the app"
- **Match CTA to scroll depth.** A high-intent visitor at the bottom can
  handle a longer label than someone who just landed.

## FAQ Section (near the bottom)

Use the FAQ to answer the **objections** that stop people from converting —
not generic questions. If you've never seen a real user ask a question, it
doesn't belong here.

Common objection categories:
- **Price / commitment.** "Is there a free version?" "Can I cancel anytime?"
- **Fit.** "Does this work for [my specific case]?"
- **Trust.** "Who's behind this?" "What happens to my data?"
- **Switching cost.** "Can I import from [other tool]?"

Answer in plain language. Lead with "yes" or "no" if it's a yes/no question.

## Final CTA Section (bottom of page)

The last section before the footer should restate the promise and give one
final CTA. This is for visitors who scrolled past everything and need a
reminder of the offer.

```
[ Restated outcome — same energy as the hero, different angle ]
[ Primary CTA ]
[ Tiny reassurance: "Free to try. No credit card." ]
```

## Things Marketers Love That You Should Cut

- **"Trusted by industry leaders"** without the actual logos.
- **"Powerful, intuitive, seamless"** — say what it does instead.
- **A list of every framework, integration, or platform** as a feature
  section. Most visitors don't care.
- **Stock photos of smiling people in offices.** Use real screenshots.
- **A signup form above the hero.** Earn the email first.
- **Multi-step heroes with carousels.** Pick the strongest message and commit.
- **"Join the waitlist" with no reason to.** Tell them what they're waiting
  for.

## Quick Checklist

- [ ] Above-the-fold answers: what / who / why / what next
- [ ] Headline is ≤10 words and outcome-first
- [ ] Subhead makes the headline concrete in one sentence
- [ ] Hand test: cover the hero image — does the text sell alone?
- [ ] Each feature section = one outcome, one feature, one image
- [ ] Section headlines are outcomes, not categories ("Features", "Benefits")
- [ ] Primary CTA repeats every 1–2 sections, with verb + outcome labels
- [ ] Social proof (if any) is real, specific, and high on the page
- [ ] FAQ answers real objections, not invented questions
- [ ] No "powerful" / "seamless" / "intuitive" / "Welcome to..." / "Introducing"
